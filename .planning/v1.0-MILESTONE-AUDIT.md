Primary E2E:

```text
config/namespace
→ UUIDv5/SHA-256
→ typed entities
→ typed edges
→ Episodic/MENTIONS/fact provenance
→ CatalogIngestBatch
→ atomic commit/retry/status
→ search/community
```

Failure E2E:

```text
preflight conflict or transaction race
→ no partial domain mutation
→ rollback
→ bounded failed status in separate transaction
```

Dry-run remains write-free. Identical committed retries remain unchanged. Catalog paths bypass LLM extraction, queueing, `add_episode`, implicit endpoint creation, and automatic community creation.

## Atomic Concurrency Closure

- Source validation and mutation use one fixed, group-scoped Neo4j compare-and-set query.

- Real concurrent live regression: exactly one conflicting source update committed; loser returned `batch_conflict`; stored hash matched one contender.
