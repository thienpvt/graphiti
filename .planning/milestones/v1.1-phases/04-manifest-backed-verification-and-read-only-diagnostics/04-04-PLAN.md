## Interface contracts

- Delete/replace batch-only branches that set `section.expected = len(rows)` (catalog_service.py ~1783-1786, ~1848-1850).
- When `request.batch_id` present: load manifest via plan-03 helper; build expected entity/edge/evidence refs from body; live MATCH = observations; diff missing vs extras.
- Keys-only (no batch_id): request keys remain expected authority; no fake manifest.
- Batch+keys: manifest expected membership for batch; keys still diagnosed against live rows.
- Use existing `CatalogErrorCode.manifest_mismatch` for missing/invalid manifest on committed catalog-v2 batch (Q3 RESOLVED / D-09). Missing batch status uses GATE-05 `found=false` path — not `manifest_mismatch`.
- Embedding presence always checked for members requiring embeddings; no optional skip flag (D-08).
- Additive response fields preferred (`extras`, `manifest_sha256`, evidence section) — no breaking renames (A3).
- EVID-13 (Q1 RESOLVED): exact evidence MATCH authority is `group_id + evidence-link uuid` from durable manifest members; compare `link_key`/`content_sha256` after load; never link_key-only fallback when uuid present; corrupt/missing uuid on member fail closed.
- Edge observation RETURN for consistency checks must include `e.content_sha256` (Q2 RESOLVED); expected hash from manifest edge member.
