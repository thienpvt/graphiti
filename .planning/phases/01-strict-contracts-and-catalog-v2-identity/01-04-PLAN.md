## CONT-07 production mechanism (LOCKED — no executor discretion)

**Chosen route:** FastMCP already type-binds each catalog tool parameter as a Pydantic request model (`request: UpsertTypedEntitiesRequest`, etc. at `graphiti_mcp_server.py` catalog tools). Framework runs `model_validate` before the tool body executes. Invalid nested data never enters wrapper/service/store/embed/schema/tx/status.

**Mandatory proof (not optional):**
1. Source/registration test asserts all seven frozen tool names exist and each binds a typed catalog request model (not bare `dict`/`Any`).
2. Invalid nested payload cases call the same request classes via `Model.model_validate(...)` (the production validation mechanism FastMCP uses) and assert ValidationError with zero CatalogService/store/embedder/schema/tx/status calls (AsyncMock spies).
3. `catalog_validation_error_to_structured` converts ValidationError to SAFE-08 shape for any catch/log path; do not invent seven duplicate wrappers unless a single catch point already exists without broad refactor.
4. Preserve seven tool names exactly: upsert_typed_entities, resolve_typed_entities, verify_catalog_batch, upsert_typed_edges, upsert_provenance, get_catalog_ingest_status, upsert_catalog_batch.

**Forbidden:** optional escape language; treating unit-only model_validate without tying it to the FastMCP typed-param production mechanism; renaming tools.

## Interface contracts

```text
catalog_validation_error_to_structured(
    exc: ValidationError,
    *,
    correlation_id: str,
) -> dict[str, Any]
# keys: code, message, field_path, retryable, correlation_id
# code: CatalogErrorCode.validation_error or mapped unsupported_identity_schema / invalid_system_key when loc/type indicates
# message: bounded <=512, non-sensitive, no str(exc) full dump
# field_path: '.'.join(map(str, loc)) or None
# retryable: False for validation
```

Map when possible:
- identity_schema_version errors → unsupported_identity_schema
- system_key errors → invalid_system_key
- else validation_error
