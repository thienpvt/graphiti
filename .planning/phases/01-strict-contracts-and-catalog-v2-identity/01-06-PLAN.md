## Locked contract details

- `CatalogSourceRef.document_id`: `None` allowed. If present, strict string, `min_length=1`, `max_length=MAX_SHORT_STRING_LENGTH`; no stripping or coercion.
- `CatalogSourceRef.page`: required strict integer, `gt=0`; bool, float, and numeric string rejected.
- `CatalogSourceRef.raw_text`: required strict string, `max_length=MAX_EVIDENCE_LENGTH`; empty accepted; trailing characters preserved exactly.
- `CatalogEntityItem.source_refs`: `list[CatalogSourceRef] | None`, explicit `max_length=MAX_SOURCE_REFS`; unknown keys and legacy `doc`/`line` shape rejected by `CatalogStrictModel`.
- `StrictTrue`: shared `Annotated[Literal[True], BeforeValidator(...), WithJsonSchema({'type': 'boolean', 'const': True})]`; before-validator accepts only `value is True`.
- `SystemKey`: shared `Annotated[Literal['FE', 'BO', 'COMMON'], BeforeValidator(...)]`; every present invalid value raises `PydanticCustomError('invalid_system_key', 'Invalid system key')` before Literal validation. Missing stays `type='missing'`; converter returns `validation_error` with `field_path='system_key'`.
- Nested system mismatch: exact request-relative paths, no leading `request.`: `entities.0.graph_key`, `edges.0.source_graph_key`, `edges.0.target_graph_key`, `entity_targets.0.graph_key`, `provenance.entity_targets.0.graph_key`, plus corresponding resolve/verify routes.
- No new package. Installed Pydantic 2.11.7 supports `BeforeValidator`, `WithJsonSchema`, `StrictInt`, `StrictStr`, `PydanticCustomError`, and `ValidationError.from_exception_data`.
