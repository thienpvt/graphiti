## Interface contracts

```text
class CatalogSourceRef(CatalogStrictModel):
    document_id: str | None  # optional; max_length=MAX_SHORT_STRING_LENGTH; empty string rejected or treated as absent only if None
    page: int                # required; gt=0 integer (reject bool via strict int path)
    raw_text: str            # required; max_length=MAX_EVIDENCE_LENGTH; preserve exact including empty and trailing spaces
    # extra=forbid via CatalogStrictModel

CatalogEntityItem.source_refs: list[CatalogSourceRef] | None
# store path: model_dump() plain JSON dicts before serialize_nested_json — no CatalogSourceRef objects in Cypher params

StrictTrue = Annotated[
    Literal[True],
    BeforeValidator(_require_literal_true),  # accept only `is True`; reject 1, 0, 'true', '1', False, None
    # JSON schema remains type boolean + const true (WithJsonSchema or equivalent)
]
# Apply to every catalog atomic and strict_endpoints field

ResolveTypedEntitiesRequest.entities: list[ResolveEntityRef] = Field(..., min_length=1, max_length=HARD_MAX_ENTITIES_PER_BATCH)

# system mismatch:
raise PydanticCustomError('invalid_system_key', 'Invalid system key')
# from validate_entity_graph_key and any nested shell/endpoint system mismatch validators
# converter: if first error type == 'invalid_system_key' → CatalogErrorCode.invalid_system_key
# field_path: request-relative sanitized loc (strip leading 'request' only)

# catalog log policy (service/wrapper catalog-specific format strings):
# allowed: batch_id, counts, codes, exception type names
# forbidden: group_id interpolation in catalog-specific log format strings
```
