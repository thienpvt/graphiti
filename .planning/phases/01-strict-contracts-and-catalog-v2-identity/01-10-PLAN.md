## Locked implementation contract

- The pre-read and `_recheck_entity_in_tx` may remain as fail-fast advisory checks; neither is mutation authority. The locking MERGE row is authoritative for create/update/unchanged/conflict.
- Under-lock immutable checks cover `name`, `graph_key`, `name_raw`, `name_canonical`, and exact fixed custom entity type/labels. A conflict returns `status='error'` and `error_code='deterministic_uuid_conflict'` before mutable/hash/vector branches.
- `created_at`, labels, immutable names, `content_sha256`, mutable payload, `batch_id`, `updated_at`, and `name_embedding` remain untouched on conflict.
- Atomic standalone conflict rolls back siblings. Per-item standalone returns an exact per-item conflict. Combined batch rolls back the complete domain transaction, then may persist only its existing safe failed-status record in the separate status transaction; it returns deterministic conflict, not generic DB failure.
- All graph keys in migrated fixtures are grammar-correct catalog-v2 FE keys. No automatic catalog-v1 migration helper is introduced.
- Every command specification is a JSON argv array run with `subprocess.run(argv, shell=False, cwd=repo_root)` from repository root. Never execute joined strings.
- TDD RED requires successful collect-only plus inner pytest exit exactly 1; every other inner exit fails the outer wrapper. Preserve RED commit ancestry.
