## Interface contracts

- Mirror `resolve_typed_entities` flow: _read_gate → store MATCH → group by key → anomaly tags → log counts only.
- Edge anomalies: missing, duplicate_edge_key, edge_type_mismatch, endpoint_mismatch, endpoint_pair_violation, uuid_mismatch, missing_embedding (align names with existing edge verify where possible).
- Evidence: MATCH CatalogEvidenceLink by group_id + target_uuid (or stored link target fields); ORDER stable; page in service with hard max; default compact no full excerpt.
- Optional include_excerpts flag length-bounded by existing MAX evidence length constants.
- Full graph keys on source/target/entity identities.
- Do not register MCP tools here (plan 06).
